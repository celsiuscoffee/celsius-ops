Design system — semantic vs brand

Terracotta (#C05040) is the brand colour, but it sits in the same red-orange family as a typical "danger" semantic. Using it for both brand action and state makes Brewing / Failed / Cancelled all read with the same hue and lose contrast. This page splits them: terracotta stays for primary action / brand chrome only; every state moment uses an explicit semantic colour (success = green, warning/in-flight = amber, danger = red, info = blue).

What changes in code

Add to Tailwind theme (in tailwind.config.js): 
colors: {
  // brand
  espresso:    '#160800',
  primary:     '#C05040',      // alias: terracotta
  terracotta:  '#C05040',
  // semantic
  success:     { DEFAULT: '#2E7D32', tint: '#E8F5E9', fg: '#1B5E20' },
  warning:     { DEFAULT: '#B45309', tint: '#FEF3C7', fg: '#78350F' },
  danger:      { DEFAULT: '#B91C1C', tint: '#FEE2E2', fg: '#7F1D1D' },
  info:        { DEFAULT: '#1D4ED8', tint: '#DBEAFE', fg: '#1E3A8A' },
}
Then replace the inline hex literals in app/order/[id].tsx and friends:
  • backgroundColor: "#E8F5E9" → bg-success-tint
  • backgroundColor: "#FFF3E0" (in-flight states) → bg-warning-tint
  • color: "#B0413E" (failed/cancelled icons) → text-danger
  • "Confirming payment" overlay disc background → bg-warning
  • "Payment successful" overlay disc background → bg-success
  • "Payment failed" / "Order cancelled" status card chip → bg-danger-tint text-danger-fg
